## Why

使用者在桌機新增信用卡時，需手動逐一輸入回饋率、特約店家、消費上限等資訊，過程繁瑣且容易出錯。透過自動擷取銀行活動網頁內容，可大幅降低手動輸入的負擔。

## What Changes

- 在新增卡片流程中新增「從活動網頁匯入」入口，讓使用者貼上銀行促銷頁面網址
- 後端（或前端）透過 AI（Claude API）解析網頁 HTML，自動抽取以下欄位：
  - 卡片名稱
  - 海外基本回饋率（%）
  - 每月上限類型（回饋上限 / 消費上限）及金額（NTD）
  - 特約店家清單（店家名稱、加碼回饋率、單店消費上限）
- 擷取結果以「預填表單」形式呈現，使用者可在儲存前確認或修改
- 擷取失敗時顯示明確錯誤訊息，讓使用者改為手動輸入

## Non-Goals

- 不自動定期抓取網頁更新（無排程爬蟲）
- 不支援需要登入才能查看的銀行會員頁面
- 不解析 PDF 或圖片格式的活動說明

## Capabilities

### New Capabilities

- `card-import-from-url`: 從銀行活動網址自動擷取信用卡回饋資訊並預填新增卡片表單

### Modified Capabilities

- `card-settings`: 新增「從網址匯入」入口按鈕及預填流程，擴充現有新增卡片表單的觸發方式

## Impact

- 受影響的規格：`card-settings`（新增匯入入口）、新增 `card-import-from-url` 規格
- 受影響的程式碼：
  - `index.html` / 卡片設定 UI 元件（新增「從網址匯入」按鈕與預填邏輯）
  - 新增擷取模組（呼叫 Claude API 解析網頁 HTML）
  - `localStorage` 卡片資料結構（不變，預填後沿用現有儲存流程）
- 外部依賴：Claude API（用於 HTML 內容解析與結構化資料抽取）
