## Summary

試算頁兩項 UX 改善：（1）自訂店家輸入時缺乏視覺確認回饋；（2）信用卡推薦卡片排版難以快速掃讀，重新設計為橫向層級布局。

## Motivation

1. **自訂店家確認感缺失**：使用者輸入加碼店家以外的店名（如「無印良品」），頁面上沒有任何視覺回饋確認該名稱會被存入消費記錄。店家欄 chip 全部不亮，造成使用者不確定刷卡後「去哪間店」的資訊是否有被記錄。實際上 `store` state 是正確的，是 UX 層面的問題。

2. **信用卡卡片排版過於密集**：回饋資訊（百分比、費率細項、回饋金額、明細）全部直向堆疊，使用者需要花時間逐行掃視。設計目標是讓「卡片名稱、回饋率、+刷卡按鈕」一眼就能看到，「回饋金額」是最重要的數字必須最突出。

## Proposed Solution

**自訂店家 chip**：當 `storeQuery` 非空且不匹配任何加碼店家時，在 chip 列顯示「目前店家：{storeQuery}」的確認 chip（以金色選取樣式呈現），作為「此名稱將被記帳」的視覺確認。清除輸入則 chip 消失。

**卡片橫向重構**：
```
┌──────────────────────────────────────────────────────┐
│推│ 聯邦吉鶴卡  [Apple Pay]   7%   │ +刷卡 │
│薦│ 基本2.5%                        │         │
│  │ AP1.5% + 店家3%    NT$2,450 ←大字│         │
│  │─────────────────────────────────│         │
│  │ 基本 1,250 | 加碼 600 | AP 600  │         │
│  │ ⚠ 警告訊息（如有）              │         │
└──────────────────────────────────────────────────────┘
```
- 左側窄色塊「推薦」僅最佳推薦卡顯示（其他卡不顯示色塊）
- 第一行：卡名 + 付款 badge + 回饋率 + `+刷卡` 按鈕（同一行）
- 第二行：費率細項（左）＋ 回饋總額 NT$XXX（右，大字綠色）
- 分隔線
- 分隔線下：各項金額（每項一組，小字，格式：「基本 1,250 | 店家加碼 600 | 行動支付加碼 600」）
- 警告訊息（如有）保留在底部

## Non-Goals

- 不新增「加碼店家」以外店家的回饋計算
- 不改動回饋計算邏輯
- 不改動其他 Tab 的 UI

## Impact

- Affected specs: `expense-tracker`（修改：自訂店家確認）、`card-advisor`（修改：卡片排版）
- Affected code:
  - `src/pages/CalcPage.tsx`（兩項改動均在此）
